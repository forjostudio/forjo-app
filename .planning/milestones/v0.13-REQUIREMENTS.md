# Requirements — v0.13 Vertical Canchas

**Milestone:** v0.13 — Vertical Canchas
**Workstream:** canchas
**Defined:** 2026-06-30
**Builds on:** v0.12 Motor de Reservas (motor de espacio compartido ya live: `spaces`/`agenda_spaces`/`appointment_spaces` + `book_slot_atomic` + EXCLUDE). NO se re-migra el core.

> **Goal:** Darle al rubro canchas una experiencia dedicada de punta a punta —configuración y reserva pública propias— construida ENCIMA del motor de espacio compartido de v0.12.

## v1 Requirements

### Vertical Canchas (VERT)

- [x] **VERT-01**: Un negocio puede setear su rubro a "canchas" y el dashboard adopta terminología y menú propios del vertical (sin "Profesionales/Equipo").
- [x] **VERT-02**: El vertical canchas resuelve sin romper los otros verticales (salud/belleza/general) ni los negocios existentes.

### Configuración de Canchas (CANCHA)

- [x] **CANCHA-01**: El dueño crea una cancha como entidad reservable con nombre, precio propio y duración fija (seteada por él), y le asocia uno o más espacios físicos.
- [x] **CANCHA-02**: El dueño edita y elimina canchas; cada cancha puede tener su propia duración (variable entre canchas).
- [x] **CANCHA-03**: La config de canchas reusa el motor de v0.12 (`spaces`/`agenda_spaces`) sin re-migrar; mapear cancha→espacios sigue acoplando la disponibilidad.

### Booking público de alquiler (ALQUILER)

- [x] **ALQUILER-01**: La página pública del negocio permite al cliente elegir una cancha + un horario disponible; el cliente NO elige duración (es la fija de la cancha).
- [x] **ALQUILER-02**: Reservar una cancha respeta la exclusión por espacio del motor (si comparte espacio con otra ocupada en horario solapado, no se puede reservar).
- [x] **ALQUILER-03**: Si el cliente quiere más tiempo del que dura una cancha, puede sacar dos turnos consecutivos (no hay duración custom).
- [x] **ALQUILER-04**: El precio mostrado/asociado a la reserva es el precio propio de la cancha.

## v2 / Future Requirements

Reconocidos pero diferidos — no entran en v0.13.

- **STAFF-CANCHAS-01**: Staff/profesionales para un negocio de canchas (add-on especial/pago) — ~1% de los casos.
- **SERVICIOS-ESPACIO-01**: "Servicios que comparten espacios" (clases de gym, equipamiento de consultorio que no puede convivir con otro servicio) — feature vertical-agnóstica que reusa el MISMO motor de espacio.
- **PRICING-FRANJA-01**: Pricing dinámico por franja horaria (peak/off-peak) para canchas.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Selección de duración custom por el cliente | Descartado por diseño: la duración la fija el dueño por cancha; si el cliente quiere más, saca dos turnos consecutivos |
| Staff/profesionales en el vertical canchas | Diferido (STAFF-CANCHAS-01); el rubro canchas no usa profesionales en el 99% de los casos |
| Generalizar "servicios que comparten espacios" | Diferido (SERVICIOS-ESPACIO-01); el motor ya lo soporta pero la UX/flujo es otro milestone |
| Re-migrar el core de booking/espacio | El motor de v0.12 es agnóstico al rubro y la duración variable ya está soportada (anti-solape por duración) — cero re-migración |
