# Quick Task 260719-j9e: TimeField numérico + días especiales multi-select siempre activo

Follow-up de UAT mobile de v0.22 (superficie agenda/booking). Dos mejoras
independientes ejecutadas juntas.

## Tarea 1 — TimeField numérico

**Problema:** los `<input type="time">` abren en mobile el reloj nativo (rueda),
que no se puede forzar a modo teclado. El usuario quiere tipear la hora directo
con teclado numérico.

**Solución:** `components/ui/time-field.tsx` — input de texto con
`inputMode="numeric"`, formateo progresivo al tipear (`0930` → `09:30`) y
normalización + clamp en blur (`9:5` → `09:05`, hora 0-23, min 0-59). Contrato
idéntico al `type=time` (`value`/`onChange` con string `"HH:MM"` o `""`). Sync
del value externo con ajuste-de-estado-en-render (patrón React 19, sin
`useEffect` para no violar `react-hooks/set-state-in-effect`).

**Sitios reemplazados (5):**
- `app/(dashboard)/agenda/agenda-client.tsx` — grilla de horarios (start/end) y
  horario especial de días especiales (excBulk start/end).
- `components/dashboard/nuevo-turno-form.tsx` — campo Hora.
Se quitó el hack `max-sm:[&::-webkit-calendar-picker-indicator]:hidden` (ya no
hay picker nativo) y el import de `Input` quedó sin usar en agenda-client → removido.

## Tarea 2 — Días especiales multi-select siempre activo

**Problema:** en mobile no hay Shift/Ctrl y el "mantener apretado y deslizar" es
incómodo. Como la selección se confirma en el panel, no hace falta un modo.

**Solución (`agenda-client.tsx`):** `handleDayClick` ahora siempre togglea el día
(quita el branch de selección única); Shift sigue haciendo rango. Se eliminó el
estado `excMulti` y su botón "Seleccionar varios"; en su lugar un botón "Limpiar"
visible solo cuando `excSel.size > 0`. Hint actualizado.
