<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser (editor client) → Server Action | El borrador viaja del cliente a `saveLandingConfig`; input no confiable |
| browser → Supabase (session) | La page fetchea con el session client (RLS por owner); el owner es un actor autenticado |
| feature flag (server env) → render del editor | `CMS_ENABLED` decide si la ruta existe |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Tampering | saveLandingConfig call desde el cliente | mitigate | El cliente NUNCA manda business_id; `saveLandingConfig` resuelve el business de la sesion e ignora el body (heredado T-13-01). El editor envia solo el config. |
| T-14-02 | Elevation | ruta del editor sin flag | mitigate | `page.tsx` hace `notFound()` como PRIMER paso si `CMS_ENABLED !== 'true'` (fail-closed, threat note d Phase 13/T-13-04); defensa en profundidad: la action re-chequea el flag primero. |
| T-14-03 | Information Disclosure | preview con datos de otro tenant | mitigate | Los 5 datasets del preview se fetchean con `.eq('business_id', business.id)` de la sesion; el preview solo compone el config del propio dueño contra sus propios datos. Sin fetch cross-tenant. |
| T-14-04 | Tampering | service-role en la superficie web | mitigate | `page.tsx` usa exclusivamente `createClient()` (session, RLS). Prohibido `createAdminClient()` (grep-verificado en acceptance). Heredado T-13-02. |
| T-14-05 | Availability | preview rompe el widget de booking | mitigate | Booking = caja negra: sin transform/overflow/scroll-timeline sobre un ancestro de `#reservar` dentro de `.frj-site` (L7); el overflow del frame va en el contenedor externo. Heredado RSV-02. |
| T-14-SC | Tampering | npm/pip installs | accept | Esta fase instala CERO paquetes nuevos (RESEARCH: sin dependencias). No hay superficie de supply-chain. |
</threat_model>
