<objective>
Desviar la escritura del editor de `landing_config` a `landing_draft` y agregar las dos acciones que
faltan —`publishLanding()` y `discardLandingDraft()`— como **copia server-side**. Más los helpers puros
que le permiten al editor derivar los 3 estados sin estado nuevo en la DB.

Purpose: es el corazón del milestone. A partir de acá **guardar deja de tener consecuencias públicas**
(PUB-03) y salir al aire es una decisión explícita del dueño (PUB-04). Las 3 acciones espejan, verbatim,
el molde owner-only que `/gsd:secure-phase 13` ya auditó: flag fail-closed → session client (anon +
cookies, RLS activa) → `business_id` de la **sesión** → gate `has_web_custom` **en la acción** → Zod
estricto → update acotado con verificación de filas afectadas. `publishLanding()` y
`discardLandingDraft()` **no reciben argumentos**: superficie de tampering = 0, y el config que sale al
aire se lee de la DB, nunca del body.

Output: 3 Server Actions, los 2 configs pasados por `page.tsx`, y `configsEqual` / `deriveEditorState`
puros y testeados. La UI completa (barra, dialogs, toasts) es 15-03: acá `web-client.tsx` solo se
re-cablea lo mínimo para que `tsc` y `eslint` queden verdes.
