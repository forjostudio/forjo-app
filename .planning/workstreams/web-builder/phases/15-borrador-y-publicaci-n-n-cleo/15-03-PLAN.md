<objective>
Convertir el editor de "guardar = publicar" en un editor con **borrador explícito**: una sola barra
sticky con `[estado] — Descartar · Guardar · **Publicar**`, un indicador de 3 estados excluyentes, el
dialog de go-live que aparece **exactamente una vez en la vida del negocio**, y el dialog destructivo
de descartar.

Purpose: es la parte que el dueño ve. Sin esto, el borrador existe pero nadie sabe que existe, y el copy
vigente ("Todo guardado", "se publican recién cuando tocás Guardar cambios") pasa a ser **mentira** —
que es exactamente lo que PUB-05 declara. El criterio de desempate ante cualquier duda de diseño acá:
**el dueño nunca debe publicar algo distinto de lo que ve en su preview** (D-04 ⇒ Publicar guarda
SIEMPRE antes de publicar, incluso si no hay cambios sin guardar).

Output: la barra publish completa, los 2 dialogs, el mapa de toasts extendido, el aviso de empty-state
corregido y el token `--warning` (la única deuda semántica de color que el proyecto todavía tenía).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dueño → editor (browser) | Superficie autenticada. Todo lo que el cliente decide (habilitar botones, mostrar dialogs) es **cosmético**: el guard real vive en las Server Actions (15-02) |
| editor → web pública | El preview muestra SIEMPRE el borrador; el link "Ver mi web" abre la página pública real. Son dos verdades distintas y la UI no las mezcla |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-12 | Tampering | los botones deshabilitados / el dialog de go-live como "gate" | **accept** | La UI **no es un control de seguridad** y este plan no introduce ninguno. Un dueño que saltee la UI y postee directo a las acciones cae en los mismos gates de 15-02 (flag, sesión, `business_id` de la sesión, `has_web_custom`, Zod estricto) y, en el peor caso, publica **su propio** borrador — que es su derecho. Riesgo residual: nulo para el Core Value |
| T-15-13 | Tampering (producto) | publicar contenido distinto del que el dueño ve (D-04) | mitigate | `handlePublish` guarda **siempre** antes de publicar. Publicar copia el borrador **de la DB**, así que sin el guardado previo el dueño podría publicar algo distinto de su preview. Gate: revisión de código — la llamada de guardado no está condicionada a "hay cambios sin guardar" |
| T-15-09 | Injection (XSS) | contenido del config renderizado en el chrome del editor (toasts, dialogs, aviso) | mitigate | Los copys son **literales**; lo único interpolado es `business.slug`, que ya viene de la sesión y lo escapa React (nada de `dangerouslySetInnerHTML`).
