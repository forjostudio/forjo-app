<objective>
Construir la capa de **persistencia segura** del CMS: un path autenticado **owner-only** para que el
dueño escriba su propio `landing_config` desde el panel — con **RLS por tenant** + **validación Zod
estricta** (reject-on-invalid) — detrás de un **feature flag** global fail-closed, sin exponerse a
clientes y sin flujo publish/go-live. Es la fundación sobre la que Phase 14 monta el editor visual.
Esta fase es **security-sensitive**: toca el invariante multi-tenant (Core Value).

Se verifica **sin UI** (test-only): unit tests del validador + extensión del test de aislamiento
anon-key al write path. Cubre **EDIT-05** (write path owner-only + Zod) y **EDIT-07** (feature flag,
CMS 100% oculto este milestone).

Purpose: sin este write path seguro, el editor de Phase 14 no tiene dónde persistir; hacerlo mal
re-abriría la fuga de secretos de v0.9 o rompería el aislamiento cross-tenant (Core Value).
Output: `lib/landing/write.ts` (validador puro) + `lib/landing/write.test.ts` (unit) +
`app/(dashboard)/web/_landing-actions.ts` (Server Action) + 2 `it` nuevos en `test/isolation.test.ts`.
Sin migración (RESEARCH Focus 1: la policy `owner access` ya habilita el owner-write), sin dependencia
nueva, cero service-role en la superficie web.
</objective>
