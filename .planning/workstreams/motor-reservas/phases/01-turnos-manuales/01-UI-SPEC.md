# Phase 1 — UI Design Contract

> Visual and interaction contract for the manual-booking surface (alta de turno desde el dashboard). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase reuses the existing dashboard design system in full.** No new tokens, fonts, colors, or libraries are introduced. All values below are pre-populated from `app/globals.css`, `components.json`, and the existing agenda/appointments surfaces (`agenda-client.tsx`, `appointments-client.tsx`). The new UI is a **modal (desktop) / drawer (mobile) form** that drops into surfaces that already exist.

## Spacing Scale

Declared values (must be multiples of 4). Mirrors the existing dashboard rhythm (`space-y-6`, `Card p-6`, `gap-2/3/4`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline chip padding (`gap-1`, `py-1`) |
| sm | 8px | Compact element spacing, field-to-field inside a row (`gap-2`) |
| md | 12px | Form row spacing inside the modal/drawer (`gap-3`, `space-y-3`) |
| lg | 16px | Default block spacing, label-to-input (`space-y-4`) |
| xl | 24px | Modal/drawer content padding, section breaks (`p-6`, `space-y-6`) |
| 2xl | 32px | Layout gaps between major regions |
| 3xl | 48px | Reserved (page-level; not expected in this form) |

Exceptions:
- **Touch targets 44px minimum** on the trigger button and form submit (mobile). Existing icon buttons use `h-8 w-8` (32px) for dense table actions; new primary actions in the form must keep `min-height: 2.75rem` / default `Button` size on mobile per WCAG AA.
- Combobox list rows: `py-1.5` (6px) vertical to match `SelectItem` density already in the codebase.

## Typography

Mirrors the dashboard scale (3 effective sizes in this form + the page display heading). No new sizes introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display (page H1, e.g. "Agenda") | 24px (`text-2xl`) | 700 (bold), Archivo | 1.2 (`-0.02em` tracking) |
| Heading (modal/drawer title) | 18px (`DialogTitle` default, `font-semibold`) | 600, Archivo | 1.2 |
| Body / input text | 14px (`text-sm`) | 400, Space Grotesk | 1.5 |
| Label / helper / meta | 12px (`text-xs`) | 500 (`font-medium`) for labels / 400 for `text-muted-foreground` helper | 1.4 |

Weights used: **400 (regular)** + **600 (semibold)**; **700** only on the page-level H1 that already exists (not introduced by this phase). Form labels use `font-medium` (500) consistent with existing `<Label>` usage.

## Color

60/30/10 split from the Forjo cream/ink theme (`app/globals.css` `:root` / `.dark`). Accent = per-business palette (`--primary`, default Rojo Forjo `#d94a2b`).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `#f3ead8` light / `#1a1714` dark | Modal/drawer surface backdrop, page background |
| Secondary (30%) | `--card` `#fbf3e3` / `#252019`, `--secondary` `#e9ddc4` / `#2e2820` | Modal/drawer card surface, combobox dropdown, "crear nuevo cliente" inline panel (`bg-secondary/50`) |
| Accent (10%) | `--primary` (`#d94a2b` default, palette-dependent) | See reserved-for list below |
| Destructive | `--destructive` `#b23a26` / `#e05c43` | NONE in this phase — manual booking has no destructive action (see Copywriting). Token listed only for completeness. |

Accent reserved for (explicit — never "all interactive elements"):
- The primary submit button "Agregar turno" (filled `--primary`).
- The "Nuevo turno" trigger button (default `Button`, filled primary) in Agenda + Turnos headers.
- The selected/hovered state of an empty grid slot when click-to-prefill is armed (`hover:border-primary`, matching the existing calendar-day hover in `agenda-client.tsx`).
- The selected client row indicator / chosen-client check (`text-primary`).
- Focus ring on inputs (`--ring` = `--primary`), inherited from base shadcn — not a net-new use.

Semantic note: the manual turno always resolves to `status: confirmed`. If any status chip is shown after creation, reuse the existing `confirmed` chip style (`bg-primary/10 text-foreground border-primary/30` in agenda, or green `confirmed` badge in appointments) — do **not** invent a new status color.

## Copywriting Contract

Spanish (project language). Microcopy follows the action-verb + context rule and the existing toast/error conventions (`sonner` + `{ ok, error }` snake_case).

| Element | Copy |
|---------|------|
| Trigger button | **"Nuevo turno"** (con icono `Plus`) — consistente con el verbo de los CTAs del dashboard |
| Modal/drawer title | **"Nuevo turno"** |
| Primary CTA (submit) | **"Agregar turno"** (estado loading: **"Agregando..."**, botón disabled para evitar doble submit, patrón `useState(loading)` del repo) |
| Client field — buscar | Placeholder: **"Buscá un cliente o creá uno nuevo"** |
| Client field — crear nuevo (inline) | Acción: **"Crear nuevo cliente"**; campos con label visible: **"Nombre"** (requerido), **"Teléfono o email"** (al menos uno) |
| Dedupe sugerencia (D-04) | **"Ya tenés un cliente con ese teléfono: {nombre}. ¿Usar el existente?"** + acción **"Usar existente"** |
| Empty state heading (combobox sin resultados) | **"Sin clientes que coincidan"** |
| Empty state body | **"Creá un cliente nuevo con el nombre y un contacto."** + acción inline "Crear nuevo cliente" |
| Fuera-de-grilla (D-06) hint | Texto auxiliar bajo fecha/hora: **"Podés agendar a cualquier hora libre, aunque esté fuera de tu horario de atención."** |
| Error — slot ocupado (`slot_taken`, 409) | **"Ese horario ya está ocupado. Elegí otro."** (toast.error) — mismo origen que el booking público |
| Error — servicio/profesional inválido (`invalid_service` / `invalid_professional`, 400) | **"Revisá el servicio o el profesional seleccionado."** |
| Error — campos faltantes (`missing_fields`, 400) | **"Completá el cliente, el servicio y el horario."** |
| Error — fallo de inserción (`insert_failed`, 500) | **"No se pudo guardar el turno. Probá de nuevo."** |
| Success | **"Turno agregado"** (toast.success) + cierre del modal/drawer + `router.refresh()` |
| Destructive confirmation | **Ninguna.** Esta fase no tiene acciones destructivas (no se borra ni cancela nada desde el alta; el turno se crea directo `confirmed`). El alta es no-destructiva por definición. |

## Interaction States

All interactive elements declare the five required states (default, hover, focus, active, disabled) via the existing shadcn `Button`/`Input`/`Select` primitives. Phase-specific notes:

| Element | Notes |
|---------|-------|
| "Nuevo turno" trigger | Default filled primary. Focus ring `--ring`. No hover-only feedback. |
| Empty grid slot (click-to-prefill, D-08) | Default: `border-border bg-card`. Hover: `hover:border-primary` (mirrors calendar-day cells). Must be a real `<button>` (semantic), focusable, with `aria-label` describing date/hour. Touch: tap opens the form pre-filled — no hover dependency. |
| Client combobox | Loading state while filtering; empty state copy above; keyboard navigable (base-ui primitive). |
| Submit button | Disabled + "Agregando..." while the request is in flight (no double submit). Inline field errors `onBlur` (Zod, repo pattern), toast for server errors. |
| Modal/drawer dismissal | Escape, click-outside, and explicit close (X) — Dialog/Drawer primitives already provide this. Focus trap inside. |

Accessibility: contrast follows the theme tokens (cream/ink already WCAG AA in the dashboard). Form labels are always visible (no placeholder-only labels). Heading hierarchy: page `h1` already present; the modal title is a `DialogTitle` (labels the dialog, not a document heading).
