**WR-01 — la pantalla descarta el número del servidor y se contradice sola.** `POST /api/abonos/cancel/[token]` devuelve `{ ok:true, cancelledCount, lastDate }` con el efecto REAL, y el cliente tira esa respuesta (`const data = await res.json()` solo se usa para ramificar). La vista `done` sigue renderizando el `cancelledCount` / `lastDate` del preview server-rendered al cargar la página. Dos síntomas: (a) si entre el render y el click alguien cancela un turno de la serie, la pantalla de éxito informa un número que nunca ocurrió — el panel sí respeta la regla ("El servidor es la autoridad del número", `abonos-client.tsx:169-171`), acá se rompe el mismo invariante D-03/D-11; (b) en la rama `already_cancelled` se pasa a `view='cancelled'` **sin limpiar el bloque de detalle**, así que la pantalla dice "Este turno fijo ya fue dado de baja" arriba y "Se cancelan N turnos futuros" abajo, al mismo tiempo.

**WR-05 (mitad ruta).** Las dos vías justifican lo contrario sobre `after()`: la ruta pública afirma que "en serverless el fetch a Resend se corta al hacer return si no se espera" y hace `await` de DOS POST a Resend en serie dentro del handler; la ruta del panel, del mismo plan, usa `after()` para el mismo mail. Las dos afirmaciones no pueden ser ciertas — `after()` existe justamente para esto en Next 16 y está soportado en Vercel. Con el `await`, una degradación de Resend cuelga la respuesta hasta el límite de la función: el usuario ve "No pudimos dar de baja el turno fijo. Probá de nuevo" **para una baja que sí se ejecutó**, y al reintentar cae en `already_cancelled`. El timeout del POST lo cierra el Plan 07-07; acá se unifica el criterio.

**WR-04 (consumo).** El Plan 07-06 hace que `previewAbonoCancellation` devuelva `unknown: true` cuando la query falla. Acá la página lo propaga y el cliente lo dice: "No pudimos calcular cuántos turnos se van a cancelar. Se dan de baja TODOS los turnos futuros de esta serie", en vez de ocultar el bloque y dejar que el cliente confirme a ciegas una acción irreversible.

**IN-04.** `metadata = { title: 'Cancelar turno fijo' }` sin `robots`. La URL contiene la credencial y la página muestra el nombre del cliente, el servicio y el horario: si el link se filtra a un lugar crawleable, un buscador puede indexar la URL con el token adentro.

**IN-05.** Los tres CTA usan `text-white` sobre `style={{ backgroundColor: accent }}`, donde `accent` es `businesses.primary_color`, editable por el dueño. Con un primario claro el botón principal de una acción destructiva queda por debajo de 4.5:1.
