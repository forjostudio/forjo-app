<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client/devtools → server action ('use server') | Una action es invocable por POST directo sin pasar por el ConfirmDialog ni el layout — entrada no confiable |
| operator → businesses/plan_prices (service-role) | Mutaciones sensibles cruzan acá: deben pasar requireAdmin + zod + audit |
| suspended business owner → dashboard / booking | Un negocio suspendido NO debe poder operar (cortar server-side, no solo en CRM) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Elevation of Privilege | server action invocada directo sin guard | mitigate | requireAdmin() PRIMERA línea de cada action (Pitfall 2); el guard del layout NO protege actions invocadas directo |
| T-02-05 | Tampering | input de action (plan/status/addon/monto fuera de rango) | mitigate | zod schema parse en cada action (enum planes/estados, addon en set fijo, priceArs int>=0) |
| T-02-06 | Repudiation | mutación sin rastro | mitigate | logAudit() service-role tras cada mutación con action code reconocido; no falsificable (Phase 1 D5) |
| T-02-07 | (integridad de negocio) | 'suspended' que marca pero no corta | mitigate | blocklist booking += 'suspended' (403) + redirect del dashboard a /suspendido (Pitfall 3); UAT del corte real |
| T-02-08 | Tampering | editar precio asumiendo que muta MP | accept | D-04: updatePlanPrice NO re-crea preapprovals; el copy avisa (Plan 04). Riesgo de negocio aceptado y documentado |
| T-02-SC | Tampering | npm installs | accept | Phase 2 no instala paquetes nuevos (RESEARCH §Package Legitimacy Audit: no aplica) |
</threat_model>
