<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anon (dueño A) → datos de B | El test ejercita la frontera real de aislamiento: una sesión anon autenticada como A intentando leer/escribir datos de B. RLS debe denegar. |
| test → proyecto Supabase dev | Las fixtures escriben/borran en el proyecto dev compartido. Teardown incompleto deja basura. |
| service-role → aserciones | Frontera anti-falso-verde: el service-role bypassa RLS; si se usa en una aserción, el test da falsa seguridad. |

## STRIDE Threat Register

> Los "threats" de esta fase son sobre la VALIDEZ DEL TEST de aislamiento: un test con service-role da un falso verde y oculta una policy rota.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Information Disclosure (test-validity) | Aserción de aislamiento con service-role | mitigate | LA trampa (Pitfall 12). Aserciones SOLO con anon-key autenticada; service-role exclusivo para seed/teardown; guard que falla ruidosamente si el cliente de aserción es service-role. |
| T-05-06 | Tampering (test-validity) | `.eq('business_id')` en la query de aserción | mitigate | Omitir `.eq('business_id')` en las queries de aserción: que RLS sea quien deniegue, no el WHERE. Un filtro pasaría aunque RLS estuviera roto. |
| T-05-07 | Tampering (test-validity) | Cross-write probado con INSERT en appointments/clients | mitigate | `appointments`/`clients` tienen `public insert WITH CHECK(true)` (booking público): un INSERT anon pasa a propósito. El test de write usa UPDATE de una fila de B; el INSERT cruzado se prueba en `services`/`business_secrets` (sin insert público). (Pitfall 6) |
| T-05-08 | Information Disclosure | Fixtures `__test_*` + usuarios auth quedan en dev tras un fallo | mitigate | Teardown en try/finally: borrar negocios (CASCADE) + `auth.admin.deleteUser` explícito; prefijo único por corrida evita colisiones (Pitfall 5). |
| T-05-09 | Spoofing (operacional CI) | Secrets de Supabase ausentes en CI → falso "todo verde" sin probar aislamiento | accept | D-03: skip graceful documentado; los webhook tests siempre corren. NOTA operacional al usuario para cargar los repo secrets (no bloqueante). El skip emite `console.warn` para no ser silencioso. |
</threat_model>
