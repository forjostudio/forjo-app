<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test → producción | Los tests importan y ejercitan código de producción (handlers de webhook). Un test mal escrito da una falsa garantía de seguridad. |
| handler → MercadoPago | El webhook llama a la API de MP por `fetch`/`mpFetch`. En tests esa frontera DEBE estar mockeada o se pega a MP real. |

## STRIDE Threat Register

> En esta fase los "threats" son sobre la VALIDEZ DEL TEST: un test roto que pasa = falsa seguridad.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering (test-validity) | `test/webhook-deposit.test.ts` mockeando el surface equivocado | mitigate | Mockear `global.fetch` (el webhook de seña usa fetch crudo, NO `mpFetch`). Si se mockeara solo `mpFetch`, el test de monto pegaría a la API real de MP (Pitfall 13). |
| T-05-02 | Repudiation (test-validity) | Verificación de firma saltada en el test | mitigate | NO mockear `@/lib/mercadopago` en el test de seña; en el de suscripción usar mock PARCIAL que preserva `verifyMPSignature` real. La firma se computa con el mismo HMAC que producción. |
| T-05-03 | Information Disclosure (test-validity) | Test de monto que confirma sin verificar el efecto | mitigate | Asertar las llamadas registradas en el admin client mockeado (`update({payment_status:'amount_mismatch'})` presente, `status:'confirmed'` ausente), no solo el status 200. |
| T-05-04 | Denial of Service (CI) | Test que pega a la API real de MP → flaky/offline en CI | mitigate | Todo `fetch`/`mpFetch` mockeado; ninguna llamada de red en los tests de webhook. CI corre sin tocar mercadopago.com. |
| T-05-SC | Tampering | npm installs (vitest + 2 plugins) | mitigate | RESEARCH §Package Legitimacy Audit: los 3 paquetes Approved (oficiales, repos verificados, sin postinstall sospechoso). `vitest` marcado SUS por `too-new` = falso positivo (paquete canónico, 70M downloads/sem). Pinear `^4.1` (no exact). NO requiere checkpoint humano. |
</threat_model>
