| session browser client → RLS del bucket | La RLS owner-write (migr. 030) exige que el primer segmento del path sea un business del owner |

| T-14-09 | Tampering / Elevation | subida cross-tenant (escribir bajo el prefijo de otro negocio) | mitigate | `buildUploadPath` construye SIEMPRE `${businessId}/...` con el `business.id` que la page paso desde la SESION (nunca slug ni id del cliente), y la seccion se sanitiza para que no inyecte segmentos; la RLS INSERT del bucket (migr. 030) rechaza cualquier path cuyo primer segmento no sea un business del owner. Unit test prueba el invariante del path. |
| T-14-10 | Tampering | service-role en el upload de la web | mitigate | El upload usa exclusivamente el browser session client (`@/lib/supabase/client`); prohibido `createAdminClient()` (grep-verificado). Heredado invariante Phase 13 (T-13-02). |
| T-14-11 | Injection / Input Validation | archivo malicioso o URL no-http en el config | mitigate | `validateImageFile` (tipo/tamaño) antes de subir; solo se escribe al draft la `publicUrl` de `getPublicUrl` (https valido); al Guardar, `z.string().url()` + Zod server-side (parseLandingConfigForWrite) es la barrera no-bypasseable (L4, heredado Phase 13). |
