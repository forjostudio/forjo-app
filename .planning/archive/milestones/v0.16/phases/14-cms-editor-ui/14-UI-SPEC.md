# Phase 14 — UI Design Contract

> Visual and interaction contract for the CMS editor UI (`app/(dashboard)/web/`). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** the layout/interaction decisions are LOCKED in `14-CONTEXT.md` (D-01..D-07). This SPEC turns those into precise visual/interaction contracts grounded in the EXISTING dashboard design system (`app/globals.css`, `settings-client.tsx`). It does **not** re-open D-01..D-07.
>
> **Non-goal:** the SPEC does not re-style the landing preview. The preview re-renders the REAL `components/landing/landing-renderer.tsx` with the draft config; its look is owned by v0.10/Phase 8.1/Phase 12, not by this editor. The editor CHROME is what this SPEC governs — it must be consistent with the dashboard, NOT with the landing.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (already initialized — `components.json`, style `base-nova`, baseColor `neutral`, CSS variables) |
| Preset | `base-nova` / `neutral` / rsc:true / iconLibrary lucide (from `components.json`) — no new registry |
| Component library | `@base-ui/react` primitives under shadcn wrappers in `@/components/ui` |
| Icon library | `lucide-react` (single system — do NOT mix icon sets) |
| Font | Chrome: `--font-sans` (Space Grotesk), headings `--font-heading` (Archivo). Preview: whatever `resolveLandingTheme` resolves — untouched by the editor |

**Reuse-first mandate (project rule 13.4 + "mirror the existing pattern"):** every editor control MUST reuse the components already in `@/components/ui` (`Card`, `Input`, `Label`, `Textarea`, `Button`, `Select`, `Separator`, `Tabs`, `Dialog`, `Drawer` [vaul], `sonner`). The theme/palette/font swatch grid MUST mirror `settings-client.tsx` (the `THEMES.map` / `THEME_PALETTES.map` button pattern, lines 852–967) verbatim in structure — only the write target changes (`landing_config.theme`, NOT `businesses.theme/palette/font`). **No new dependency.** There is no `switch` component in `@/components/ui`; the `enabled` toggle uses the segmented-button pattern already used for the light/dark toggle in settings (lines 977–999), not a new Switch.

## Spacing Scale

Declared values (all multiples of 4 — matches Tailwind default + settings-client usage `space-y-6 / gap-3 / p-6`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, inline chip padding (`gap-1`) |
| sm | 8px | Compact control spacing, swatch inner gaps (`gap-2`) |
| md | 12px | Default gap between form fields and grid cells (`gap-3`, `space-y-3`) — settings baseline |
| lg | 16px | Field group spacing, card inner padding on mobile (`space-y-4`) |
| xl | 24px | Card padding + section separators (`p-6`, `space-y-6`) — settings baseline |
| 2xl | 32px | Editor↔preview column gap on desktop (`gap-8`) |
| 3xl | 48px | Empty-state vertical padding |

Exceptions:
- **Touch targets ≥ 44px** for icon-only controls (up/down reorder arrows, image delete/replace buttons, mobile Editar/Preview toggle). Use `min-h-11 min-w-11` (44px) — non-negotiable (CLAUDE.md mobile rule). This overrides the visual density of the icon glyph, not the spacing scale.
- Preview panel internal spacing is owned by the landing renderer (`.frj-sec` clamp() padding) — the editor does not impose the scale inside the preview iframe-less container.

## Typography

Chrome typography only (the preview uses the landing's own type scale). Mirrors `settings-client.tsx`: page `h1` = `text-2xl font-bold` + `--font-heading`; section titles = `font-semibold text-sm`; helper text = `text-xs text-muted-foreground`; meta = `text-[10px]`.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Page title (h1) | 24px (`text-2xl`) | 700 (`font-bold`) | 1.2 | `font-[family-name:var(--font-heading)]` — "Tu web" |
| Section heading | 14px (`text-sm`) | 600 (`font-semibold`) | 1.4 | Section-panel headers ("Hero", "Nosotros"…) + control-group titles |
| Body / input | 16px (`text-base`, shadcn `Input` default) | 400 | 1.5 | Form field values, textarea copy — never < 16px on mobile (CLAUDE.md) |
| Helper / meta | 12px (`text-xs`) | 400 | 1.4 | Field hints, "(opcional)", char counters, error hints |

**Weights declared: exactly 2** — regular (400) and semibold/bold (600/700). Do not introduce a third weight. **Sizes declared: 4** (24 / 16 / 14 / 12) matching the settings surface exactly.

## Color

All colors come from CSS tokens in `app/globals.css` (Bauhaus, dark by default in the dashboard shell). **No hardcoded hex in the editor chrome** (CLAUDE.md rule) — use the `bg-*/text-*/border-*` token utilities. The palette swatch previews are the ONLY place raw hex appears, and those hex come from `THEME_PALETTES` (data), rendered as inline `style={{ backgroundColor }}` exactly like settings — that is chrome-of-a-swatch, not app color.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (`#1a1714` dark) | Editor page background, preview gutter |
| Secondary (30%) | `--card` / `--secondary` (`#252019` / `#2e2820` dark) | Editor panels, section rows, form cards, the section-list column |
| Accent (10%) | `--primary` (per-business palette; default `#e85c3f` dark red) | Active swatch ring, "Guardar cambios" primary button, active tab underline, active toggle fill, unsaved-changes dot |
| Destructive | `--destructive` (`#e05c43` dark) | Image delete button, "Descartar cambios" in confirm-on-exit dialog, error toasts |
| Muted | `--muted-foreground` (`#a99e8b` dark) | Helper text, disabled section labels, ghost-index hint |
| Border/ring | `--border` / `--ring` | Card borders, focus rings (`focus-visible:ring-2 ring-ring`) |

Accent reserved for: **the single primary Save action ("Guardar cambios"), the active-selection indicator on theme/palette/font/motion swatches (`border-primary ring-2 ring-primary/20`, mirroring settings), the active editor tab, the enabled-toggle "on" fill, and the unsaved-changes dot.** Accent is NOT used for reorder arrows, secondary buttons, image tiles, or field borders — those stay neutral (`border`, `secondary`, ghost variants).

**Contrast (WCAG AA, non-negotiable):** all token pairs above already meet AA in both modes (verified in globals.css comments, e.g. CRM primary ~8.9:1). Any NEW text-on-color the editor introduces must be checked against 4.5:1 (normal) / 3:1 (large). Do not put muted text on card without confirming ratio.

**Focus:** every interactive element gets a visible focus ring — reuse the settings pattern `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring`. Reorder arrows and toggles are `<button>`, never clickable `<div>`.

## Component Contracts

### 1. Editor shell (D-01) — split editor ↔ live preview

- **Desktop (≥ 1024px, `lg`):** two-column CSS Grid. Left = editor panel (scrollable, `overflow-y-auto`), fixed/comfortable width (~`minmax(360px, 460px)` or `lg:grid-cols-[minmax(380px,440px)_1fr]`); right = live preview (`1fr`), sticky within viewport height so it stays visible while the editor scrolls. Column gap = 32px (2xl).
- **Preview surface:** renders the REAL `LandingRenderer` with the DRAFT config + the once-server-fetched static data (business/services/professionals/timeBlocks/exceptions/locations). Client-side WYSIWYG, **no iframe, no new preview route**. Wrap the preview in a neutral framed container (`rounded-lg border bg-background overflow-hidden`) so its full-bleed `.frj-site` reads as "a page inside a frame".
- **Booking = black box inside the preview** (threat note): the preview shows the booking section as-is; the editor never wraps the booking widget in transform/overflow/scroll-timeline (Pitfall 4/8). The RSV strip IS editable (its `rsvData`), the booking widget itself is not exercised.
- **Mobile (< 1024px):** collapse to a single column with a top **segmented Editar / Preview toggle** (2-button segmented control, mirror settings light/dark toggle at lines 977–999; `min-h-11` targets). Default view = **Editar**. Switching to Preview shows the full-width live render. `vaul` Drawer is allowed for per-section edit forms on mobile if the planner prefers a bottom-sheet over inline expand (Claude's discretion, D-05 note) — but the toggle is the primary responsive mechanism, not a substitute.
- **States:** the shell has no loading state of its own (data is server-fetched by `page.tsx`); the preview updates synchronously/near-synchronously from the in-memory draft. Debounce preview re-render if needed (Claude's discretion, D-01 note) — target < 100ms perceived.

### 2. Section-list panel (D-04) — 8 fixed rows

- Renders **all 8 fixed sections ALWAYS**, in `order`: hero, about, services, gallery, location, hours, cta, booking. Never add/remove rows.
- **Row anatomy (left→right):** drag-handle-less grip icon (decorative `GripVertical` at reduced opacity, NOT interactive — reorder is via arrows) · section label (Spanish, see Copywriting) · spacer · **up button** (`ChevronUp`) · **down button** (`ChevronDown`) · **enabled toggle**. Row = `Card`-like or bordered `flex items-center gap-3 p-3 rounded-md border bg-secondary`.
- **Up/down buttons:** icon-only, `min-h-11 min-w-11`, `variant="ghost"`. First row's up is `disabled`; last row's down is `disabled` (visible disabled state, `opacity-50 cursor-not-allowed`, `aria-disabled`). Clicking rewrites `order` on the two swapped sections. Announce reorder to AT via `aria-live="polite"` region ("Sección movida a la posición N").
- **Enabled toggle:** segmented on/off or a labelled toggle-button showing `Eye`/`EyeOff` (lucide, already imported in settings). "On" = accent fill; "off" = muted. `aria-pressed` reflects state. **`booking` (and optionally `hero`) render the toggle in a locked/always-on state** (D-04b): disabled toggle pinned "on" with a tooltip/hint "Sección núcleo — siempre visible". Planner confirms the exact locked set; default recommendation = lock `booking` only.
- **Expand-to-edit:** clicking the row label/body opens that section's copy form (inline accordion on desktop, `vaul` Drawer on mobile). One section open at a time is fine. Disabled (`enabled:false`) sections still expand for editing (you can edit copy while hidden). A disabled section's label gets `text-muted-foreground`.
- **Reorder is NOT drag-and-drop** (locked D-04) — no new dep.

### 3. Per-section copy forms (D-05)

One form per section, driven by the existing per-section `data` schemas in `lib/landing/schema.ts`. Field set per section (labels in Spanish, see Copywriting), each field = `Label` + `Input`/`Textarea`, one field per line (mobile), max 2-up on desktop for short related fields:

| Section | Editable fields (from schema) | Read-only / derived | Images |
|---------|-------------------------------|---------------------|--------|
| hero | `headline`, `kicker`, `subhead`, `cta_label` (Inputs); | business name is fallback | `hero.image` (single, §4) |
| about | `title` (Input), `body` (Textarea) | — | `about.image` (single, §4) |
| services | `title`, `subtitle` (Inputs) | **services list = read-only** (from `services` table) — show a read-only preview list + a note linking to Servicios | — |
| gallery | `title` (Input) | — | `gallery.images[]` (multi-grid, §4) |
| location | `title` (Input), `map_url` (Input, URL), `show_address` (toggle) | **locations list = read-only** (from `locations` table) | — |
| hours | *(no copy — derives from `time_blocks`)* → show an informational read-only panel: "Los horarios se editan en Negocio → Horarios" | derived | — |
| cta | `headline` (Input) | — | — |
| booking | `rsvData.header` (Input), `rsvData.intro` (Textarea) | booking widget = black box (not edited) | `rsvData.images[]` (RSV gallery multi-grid, §4) |

- **Labels always visible** (never placeholder-only — CLAUDE.md form rule). Optional fields marked `(opcional)` in `text-xs text-muted-foreground` (mirror settings `ProFields`).
- **URL fields** (`map_url`) validate as https URL on blur (inline error, immediate). A non-URL shows an inline error message under the field — do not wait for Save.
- **Read-only derived panels** (services/locations/hours) use `bg-muted/40` + `text-muted-foreground` and an explicit hint pointing to where that data is edited, so the owner understands why it's not editable here.
- Every keystroke updates the in-memory draft → preview reflects it (D-03).

### 4. Image upload control (D-02) — single + RSV/gallery multi-grid

Browser-direct upload to Supabase Storage `landing-assets/{business_id}/` via the **session browser client** (`@/lib/supabase/client`), RLS owner-write. Public URL is written into the draft config and persisted only on Save (D-02b). Mirror the upload flow already in `settings-client.tsx` (`handleLogoSelect` + `uploadLogo`, lines 311–346) for validation, states, and toasts.

- **Validation (before upload):** size ≤ 2MB and type ∈ {jpeg, png, webp} — reuse settings' `validatePhoto` copy exactly ("El archivo no puede superar 2MB", "Formato no soportado. Usá JPG, PNG o WebP"). Reject inline via toast; do not enter uploading state.
- **Single-image control (hero/about):** a drop/select zone (`ImageIcon` + "Subir imagen" / drag-drop) when empty; when set, shows a thumbnail with **Reemplazar** and **Quitar** (destructive) buttons. `Reemplazar` = upload a new object, point URL at it (does not delete old object — orphan cleanup deferred, D-02c). `Quitar` = remove the URL from config only (config-only, benign orphan).
- **Multi-image grid (gallery.images / rsvData.images):** responsive grid of thumbnail tiles (`grid grid-cols-3 sm:grid-cols-4 gap-3`), each tile square (`aspect-square`, `object-cover`, `rounded-md`) with a per-tile delete (`Trash2`, top-right, `min-h-11 min-w-11` hit area, destructive on hover/focus). A trailing "+ Agregar" tile opens the file picker (multi-select allowed). Order of `images[]` = display order; per-tile up/down or delete only (no DnD).
- **States (per control):**
  - *default/empty:* dashed-border drop zone, `ImageIcon`, label + "JPG, PNG o WebP · máx 2MB".
  - *uploading:* the target tile/zone shows a spinner + disabled interaction; button text → "Subiendo…"; block Save while any upload is in flight (disable "Guardar cambios").
  - *replace:* same uploading state on the existing thumbnail.
  - *error (failed upload / oversize / wrong type):* toast with the specific reason (see Copywriting) + tile returns to previous state; the config is NOT mutated on failure.
- **Preview:** a newly-uploaded image's public URL flows into the draft → the live preview shows it immediately.
- **Security surface (threat note):** all uploads write under `{business_id}/` only; the browser client + bucket RLS enforce the prefix. Never service-role. Never construct a path outside `${business.id}/`.

### 5. Theme / palette / primary / motion controls (D-06)

Mirror `settings-client.tsx` swatch grids (lines 852–967) **structurally**, but write to `landing_config.theme` (`preset` + `overrides.palette` + `overrides.primary`) — NOT to `businesses.theme/palette/font`. Applied live in the preview via `resolveLandingTheme` (the editor computes the resolved theme and feeds the renderer / sets the preview container's data-attributes).

- **Preset (theme) grid:** reuse `THEMES` cards (bg/fg/chips mini-preview), active state = `border-primary ring-2 ring-primary/20` + `Check` badge (verbatim settings). Selecting a preset resets palette to that preset's default (`THEME_DEFAULT_PAL`) — same behavior as `selectTheme`. Writes `theme.preset`.
- **Palette grid:** reuse `THEME_PALETTES[preset]` swatch cards. Writes `theme.overrides.palette`.
- **Primary color:** a color input (native `<input type="color">` + a text field for the hex) validated against `isSafeColor` (allowlist `#` + 3/4/6/8 hex) from `lib/landing/theme.ts`. An invalid hex shows an inline error and is NOT written (the renderer would drop it anyway). Writes `theme.overrides.primary`.
- **Motion:** a **3-way segmented control** (`none` / `subtle` / `premium`) — reuse the light/dark segmented pattern, 3 buttons. Labels Spanish: "Sin movimiento" / "Sutil" / "Premium". `aria-pressed` on active; active = accent fill. Writes `config.motion`. Preview reflects it via `data-motion` on the `.frj-site` container.
- **A11y note:** the color input must have a visible `Label`; motion segmented buttons are real `<button>`s in a `role="group"` with an accessible group label ("Nivel de movimiento").

### 6. Save bar (D-03)

- **Placement:** a sticky bottom (or top-of-editor) action bar within the editor panel, always visible (Save must never be behind scroll — CLAUDE.md form rule).
- **Unsaved indicator:** when the draft differs from the last-saved config, show an accent dot + "Cambios sin guardar" (`text-xs`, accent). When clean, show "Todo guardado" (`text-muted-foreground`) or nothing.
- **"Guardar cambios" button:** primary variant, accent. States:
  - *default:* enabled only when there are unsaved changes AND no upload is in flight; otherwise `disabled` (`opacity-50`, `aria-disabled`).
  - *saving:* disabled + text → "Guardando…" + spinner (prevents double-submit, mirror settings `savingBiz` pattern).
  - *disabled:* no changes / upload in flight.
- **On result (D-03c):** call `saveLandingConfig(fullConfig)` and map the return:
  - `{ ok: true }` → `toast.success("Cambios guardados")` + clear the unsaved flag + set draft-as-saved baseline.
  - `{ ok: false, error }` → `toast.error(...)` mapped per error code (see Copywriting table). Do NOT clear the unsaved flag.
- **Confirm-on-exit (D-03b):** if the user navigates away / closes with unsaved changes, intercept (router + `beforeunload`) and show a confirm dialog (`Dialog`) — "Tenés cambios sin guardar" with **Descartar cambios** (destructive) / **Seguir editando** (default). No draft table; the draft is purely in-memory.

### 7. Empty state (business with `landing_config === null`)

- If the business has no `landing_config` (legacy / never opted in), the editor **seeds the in-memory draft from `DEFAULT_LANDING_CONFIG`** (hero + booking, `theme.preset: 'default'`, no motion) and shows a one-time inline notice at the top of the editor: heading + body + implicit next step (start editing). This is the entry into the editor, not a blank screen (CLAUDE.md empty-state rule).
- The preview immediately renders that default config (hero + booking) so the owner sees a real starting point.
- Nothing is persisted until the owner hits "Guardar cambios" — the null→default seeding is client-side only (matches D-03).

## Responsive & Accessibility Contract

- **Mobile-first, base 375px** (CLAUDE.md): single column + Editar/Vista previa toggle; forms one field per line; image grid `grid-cols-3`.
- **Breakpoints:** split editor↔preview activates at `lg` (1024px). Below that = stacked/toggle.
- **Touch targets ≥ 44px** on all icon-only controls (reorder arrows, image delete/add, toggle, mobile view switch).
- **Focus:** every control has a visible focus ring (`focus-visible:ring-2 ring-ring`); no clickable `<div>` — reorder/toggle/tiles are `<button>`.
- **Semantics:** section list is a `<ul>`/`<li>` or list role; per-section forms use `<label for>`/`Label`; reorder announces via `aria-live="polite"`; motion/toggle groups use `role="group"` + group label; disabled controls set `aria-disabled` + visible disabled style.
- **States on every interactive element:** default / hover / focus / active / disabled (CLAUDE.md).
- **Preview a11y:** the preview is the real landing markup (already AA-audited in v0.10); the editor does not degrade it. Booking stays a black box (no wrapping transform/overflow).
- **`prefers-reduced-motion`:** editor chrome transitions ≤ 200ms and only `transform`/`opacity`; the preview's motion respects `prefers-reduced-motion` via the existing landing CSS (unchanged).
